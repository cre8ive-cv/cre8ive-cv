TL;DR

  • Resume content is processed entirely in memory for previews/exports and never written to persistent storage.
  • We only retain anonymous usage metadata (e.g., timestamp, selected theme, and UI mode) to keep improving the experience.
  • Matomo analytics runs in zero-cookie mode with IP anonymization so we can gauge traffic trends without profiling visitors.

By using this service, you acknowledge and accept these terms and our data processing practices.

1. What Data We Process

When you export data, we temporarily process the following personal data:

  • Resume content: Name, contact information (email, phone, location), work experience, education, skills, projects, professional bio
  • Photos: Photography which you upload (processed as base64-encoded images)
  • Custom preferences: Selected theme, color scheme, section customizations

2. How We Process Your Data

PDF Preview and Download:

  • When you click "PDF Preview & Download", your data is sent to the server
  • A PDF buffer is generated entirely in memory using Puppeteer (headless browser) — no resume content or PDF output is written to disk
  • For preview: The in-memory PDF buffer is streamed to your browser and discarded as soon as it has been sent
  • For export: The same in-memory buffer is streamed as a download and then immediately released; nothing persists on the server once the response finishes
  • Usage statistics & customizations: When you export as PDF, HTML, or JSON, we collect and store metadata including theme, color palette, and custom names of resume sections (e.g., if you rename "Experience" to "My Work History"). Unlike the actual content of your resume (which is deleted immediately), these custom section titles are stored in our analytics database to help us understand how users customize their templates.

Privacy Guarantee: We do NOT store your resume data permanently. All processing is temporary and data is deleted immediately after use. Only metadata is collected.

3. Data Retention

  • Resume data: Processed in-memory only, never written to permanent storage
  • PDF buffers: Generated in RAM for a fraction of a second and released immediately after the preview/download response completes (no temporary files)
  • Anonymous usage statistics: Anonymous usage statistics (timestamp, theme preferences, display mode, custom section names and order) collected during PDF, HTML, or JSON exports are stored in a local SQLite database and retained indefinitely for service improvement. This database contains only usage metadata. There are NO personal information or resume content stored in the database.
  • Matomo analytics: Web analytics data is retained according to our analytics retention policy
  • Local CLI exports: If you use the optional open-source CLI, the generated HTML/PDF files are saved only to the folders you configure on your own machine (never uploaded to us). Managing or deleting those local files is your responsibility.

4. Analytics & Tracking

Matomo Web Analytics:

This website uses Matomo (self-hosted analytics) to understand how visitors use our service and improve it. We've configured Matomo with privacy in mind:

  • No tracking cookies: Matomo collects usage data without storing any analytics cookies in your browser
  • IP anonymization: IP addresses are anonymized (1 byte masked) before processing
  • Approximate location only: Only country-level location data based on anonymized IP
  • Page views and navigation: Pages visited and time spent on the site
  • Interaction data: Links clicked and downloads, along with referrer information
  • Technical information: Browser type, operating system, and screen resolution

These services may set strictly necessary security cookies to verify that a request comes from a legitimate user. Cloudflare does not use these cookies for advertising or cross-site tracking, and we do not access or store their contents.

Export Usage Statistics (SQLite Database):

Whenever you export your resume (PDF, HTML, or JSON), we collect and store anonymous usage statistics in a local database to improve the user interface and understand which features are most popular. The following metadata is stored:

  • Timestamp: Local date and time of the export (in format YYYY-MM-DD HH:MM:SS)
  • Application version: Version number of cre8ive.cv
  • Page URL: The URL you accessed the service from
  • Export type: Whether you exported as PDF, HTML, or JSON
  • Display mode: Interface theme used (dark/light/hacker mode)
  • Theme and color selections: Which resume theme and color palette you chose
  • Watermark preference: Whether the watermark was enabled or disabled (boolean value)
  • Section configuration: Which resume sections were enabled and their custom names (section names only, not the actual content)

Important: This data is completely anonymous and stored locally on our server. It is NOT linked to you, your identity, or your resume content. We do NOT store your name, contact information, resume content, or any personally identifiable information. This database only contains usage preferences and helps us understand UI/UX patterns to improve the service for all users.

Important Note regarding Custom Section Names:

We store the text you enter as Section Titles (e.g., "Education", "Projects"). Please DO NOT enter personal data (such as your name, phone number, or address) in the section titles. If you choose to enter personal information in these structural fields, be aware that it will be stored in our analytics database.

Clarification: Aside from the Section Titles described above, we do NOT store the actual content of your resume (bullet points, descriptions, dates, personal bio). That content remains strictly temporary.
